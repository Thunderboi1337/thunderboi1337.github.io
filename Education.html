CRC_Checksum_Verify-er-

This is an assignment that involved creating a C program to ensure data integrity using checksums. I implemented a method to generate a checksum for a given message, then check if the data was altered by comparing the original checksum to one generated after potential modification. It was a practical assingment in learning how to protect data against errors during transmission or storage.
Note: This was one of the first hard assignments we got in school and i'm still very proud of the effort put into to it. At this time we had only been doing C for a little more than a month.

Uncopyable Buffer

This is an assignment that focused on implementing a templated circular buffer in C++ and testing its functionality through a series of unit tests. The buffer supported adding and removing elements while ensuring that data was managed correctly even in a constrained storage scenario. Through this project, I learned about generic programming in C++, as well as the importance of testing to ensure the reliability of software components.

2023

Debouncing_and_TDD

This is an assignment that implements and tests a button debounce mechanism using the Test-Driven Development (TDD) approach. It includes setting up the hardware and software to read a button's state accurately, filtering out the noise from rapid pressing and releasing, to ensure stable and reliable button state detection. The project uses unit tests to verify the functionality of the debouncing algorithm, ensuring that the button's state changes are accurately captured and reported, even in the presence of potential electrical noise or rapid user interactions.
